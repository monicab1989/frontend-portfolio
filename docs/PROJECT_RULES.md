# Portfolio Development Rules

## Stack

- React 19
- Vite
- React Router DOM
- React Icons
- Framer Motion
- CSS Modules or separated CSS files
- Vercel deployment ready

## Code Style

- Use functional components only.
- Use ES modules.
- Keep components small and reusable.
- Avoid duplicated code.
- Use semantic HTML.
- Prioritize accessibility.

## Styling

- Do not use inline styles.
- Create a CSS file for each component.
- Use CSS variables for colors, spacing and typography.
- Use rem units for spacing, typography and layouts.
- Use px only for borders, shadows and small visual details.
- Mobile-first responsive design.

## Routing

- Use React Router DOM.
- Keep routes centralized.
- Prepare the application for future page expansion.

## Icons

- Use React Icons.
- Prefer Font Awesome, Heroicons or Tabler icons.

## Animations

- Use Framer Motion.
- Keep animations subtle and professional.
- Avoid excessive motion.

## Portfolio Goal

Build a modern, elegant and professional web developer portfolio.

Sections:

- Hero
- About Me
- Skills
- Projects
- Experience
- Contact
- Footer

The portfolio must look modern, clean, responsive and optimized for performance.
